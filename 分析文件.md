## 1. 系統架構總覽

```
單一 index.html 檔案（HTML + CSS + JS 全部內嵌）
│
├── 前端狀態層（JavaScript 變數）
│   └── 頁面載入時從 localStorage 讀入
│
├── 本地持久化層（localStorage）
│   └── 所有題庫、紀錄、設定存於瀏覽器
│
└── 雲端同步層（Google Apps Script）
    └── 單一 API Endpoint（GET 讀取 / POST 寫入）
```

**外部依賴：**

| 套件 | 來源 | 用途 |
|------|------|------|
| Chart.js | `cdn.jsdelivr.net` | 雷達圖 / 折線圖 |
| Gemini API | `generativelanguage.googleapis.com` | AI 導師解析、AI 自動出題 |
| Google Apps Script | 自訂 Endpoint | 雲端資料備份與同步 |

## 2. 資料結構

### 2.1 localStorage 鍵值一覽

| 鍵名 | 型別 | 說明 |
|------|------|------|
| `customQuizData` | `Array<Question>` | 使用者自訂題庫 |
| `officialQuizData` | `Object<session, Array<Question>>` | 官方考古題，以屆次為 key |
| `wrongAnswerHistory` | `Array<Question>` | 答錯過的題目（弱點題庫）|
| `favoriteQuestions` | `Array<Question>` | 使用者收藏的題目 |
| `quizHistory` | `Array<HistoryRecord>` | 每次測驗的分數紀錄 |
| `userStats` | `Object` | 總作答數、本日作答數、最後作答日期 |
| `geminiApiKey` | `string` | 使用者的 Gemini API 金鑰 |

### 2.2 資料型別定義

**Question（題目）**
```json
{
  "subject": "科目1：人工智慧基礎概論",
  "question": "題目文字",
  "options": ["選項A", "選項B", "選項C", "選項D"],
  "answer": "正確的選項文字（必須完全等於 options 中的其中一個）"
}
```

**HistoryRecord（測驗紀錄）**
```json
{
  "score": 80,
  "mode": "normal",
  "date": "2026/6/10 下午3:00:00"
}
```

**userStats（作答統計）**
```json
{
  "totalAnsweredCount": 150,
  "todayAnsweredCount": 20,
  "lastAnswerDate": "2026/6/10"
}
```

## 3. 雲端同步機制（核心）

### 3.1 架構概覽

系統使用 **Google Apps Script (GAS)** 作為免費的雲端儲存後端。  
所有裝置共用同一個 GAS Endpoint，透過 `no-cors` 模式的 `fetch` 呼叫。

```
瀏覽器 localStorage  ←→  JavaScript 變數  ←→  Google Apps Script API
                                                        ↕
                                               Google Sheets / Properties
```

### 3.2 API Endpoint

```javascript
const CLOUD_API_URL = "https://script.google.com/macros/s/.../exec";
```

- **GET** `CLOUD_API_URL` → 讀取所有雲端資料，回傳 JSON 物件
- **POST** `CLOUD_API_URL` + `{ key, value }` → 寫入單一鍵值

### 3.3 `syncToCloud(key, value)` — 寫入雲端

```javascript
async function syncToCloud(key, value) {
    await fetch(CLOUD_API_URL, {
        method: 'POST',
        mode: 'no-cors',          // 跨域不需要 CORS header
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ key: key, value: value })
    });
}
```

**特性：**
- 使用 `no-cors` 模式，無法讀取回應內容，屬於「fire and forget」
- 每次資料變動（答題、匯入、收藏、清除）都會觸發對應欄位的同步
- 若失敗只會在 console 印出錯誤，**不會通知使用者**

**觸發時機：**

| 操作 | 同步的 key |
|------|-----------|
| 每次答一題 | `userStats` |
| 答錯一題（新題） | `wrongAnswerHistory` |
| 切換收藏 | `favoriteQuestions` |
| 測驗結束 | `quizHistory` |
| 匯入自訂題庫 | `customQuizData` |
| 匯入官方題庫 | `officialQuizData` |
| 清除紀錄/錯題/最愛 | 對應 key（空陣列）|

### 3.4 `syncFromCloud()` — 從雲端讀取（智慧合併）

頁面載入時自動執行一次。每個資料欄位都有各自的合併策略：

#### 官方題庫（`officialQuizData`）

```
策略：以屆次為單位，誰多留誰
```

```
for each session in cloudData:
    if 本地沒有此屆次 or 本地題數 < 雲端題數:
        → 以雲端覆蓋本地

for each session in localData:
    if 雲端沒有此屆次 or 本地題數 > 雲端題數:
        → 回推更新雲端
```

> 目的：確保題庫只增不減，換裝置後不會遺失題目。

#### 自訂題庫（`customQuizData`）

```
策略：誰的題數多，以多的為準
```

```
if 雲端題數 > 本地題數 → 以雲端覆蓋本地
if 本地題數 > 雲端題數 → 回推更新雲端
```

#### 錯題本（`wrongAnswerHistory`）、我的最愛（`favoriteQuestions`）

```
策略：同上，誰多留誰
```

#### 測驗紀錄（`quizHistory`）

```
策略：雙向合併 + 去重 + 排序
```

```
1. 合併本地與雲端兩份紀錄
2. 以 (date + score + mode) 為複合 key 去重
3. 依分數降冪、日期降冪排序
4. 若合併後筆數 > 本地 → 更新本地
5. 若本地筆數 > 雲端 → 回推雲端
```

> 這是六種資料中唯一做真正「雙向合併」的欄位，其餘皆為「取最大值」策略。

#### 作答統計（`userStats`）

```
策略：分欄位取最大值，並考慮日期跨日情境
```

```
totalAnsweredCount:
    取 max(本地, 雲端)

todayAnsweredCount:
    if 雲端.lastAnswerDate === 本地.lastAnswerDate:
        取 max(本地, 雲端) 的今日數
    else if 雲端的日期比較新:
        以雲端的 today 數為準（代表雲端有更新的資料）
    else:
        本地日期較新，不動本地數值
```

### 3.5 `forceSyncFromCloud()` — 強制雲端覆蓋本地

由首頁「☁️ 雲端強制同步」按鈕觸發。

```
⚠️ 警告彈窗 → 使用者確認
        ↓
GET 雲端所有資料
        ↓
逐一無條件覆蓋本地 localStorage
（officialQuizData / customQuizData / wrongAnswerHistory /
  favoriteQuestions / userStats / quizHistory）
        ↓
重新渲染所有 UI 計數
```

**適用場景：** 換新電腦或清除瀏覽器後，需要從雲端還原所有資料。

### 3.6 同步時序圖

```
頁面載入
    │
    ├─ checkDateReset()       ← 若跨日，重置本日計數
    ├─ updateAnswerStatsUI()  ← 顯示本地統計
    ├─ startExamCountdown()   ← 啟動考試倒數
    ├─ updateDatabaseStats()  ← 顯示本地題庫數量
    ├─ updateOfficialStats()  ← 顯示官方題庫數量
    ├─ updateDynamicCounts()  ← 顯示錯題/最愛數量
    └─ syncFromCloud()        ← 非同步：與雲端比對合併
            │
            └─ 合併完成後再次呼叫 updateDynamicCounts()
                              updateDatabaseStats()
                              updateOfficialStats()
```

### 3.7 同步的限制與風險

| 問題 | 說明 |
|------|------|
| **無衝突解決** | 策略是「取多的」，若在不同裝置刪除了題目，下次同步會被舊資料覆蓋回來 |
| **no-cors 盲寫** | POST 成功與否無法判斷，靜默失敗不通知使用者 |
| **單點 GAS** | 所有裝置共用同一帳號的 GAS，沒有多使用者隔離 |
| **無版本號** | 若題目格式變更，舊版資料不會自動轉換 |
| **答題同步頻率高** | 每答一題就呼叫一次 `syncToCloud('userStats', ...)`，流量稍高 |

## 4. 測驗模式邏輯

### 4.1 六種模式比較

| 模式 | 題目來源 | 計時 | 答題反饋 | 題數 |
|------|---------|------|---------|------|
| 正常模式 | 自訂題庫 | 無 | 即時顯示對錯 | 自訂 |
| 生存模式 | 自訂題庫 | 30 秒倒數，答對 +3s，答錯 -5s | 即時顯示對錯 | 自訂 |
| 弱點特訓 | `wrongAnswerHistory` | 無 | 即時顯示對錯 | 自訂 |
| 我的最愛 | `favoriteQuestions` | 無 | 即時顯示對錯 | 自訂 |
| iPAS 模考 | 自訂題庫（隨機50題）| 60 分鐘 | **隱藏**，交卷後才顯示 | 固定 50 |
| 官方考古題 | `officialQuizDatabase[屆次]` | 60 分鐘 | **隱藏**，交卷後才顯示 | 固定 50 |

### 4.2 分數計算

```
每題分數 = 100 / 總題數
最終分數 = Math.round(累積得分)
```

### 4.3 答題流程

```
loadQuestion()
    └─ 顯示題目與選項按鈕
            │
    使用者點選選項
            │
    checkAnswer(selected, correct, btn)
            ├─ 更新 userStats（+1 作答）→ syncToCloud
            ├─ 若答錯 → 加入 wrongAnswerHistory → syncToCloud
            ├─ 生存模式：答對 +3s / 答錯 -5s
            └─ 顯示「下一題」按鈕
                        │
            document.getElementById('nextBtn').onclick
                        ├─ currentQuestionIndex < 總題數 → loadQuestion()
                        └─ 全部作答完 → showResult(false)
```

### 4.4 結算畫面

`showResult()` 執行內容：
1. 計算並顯示最終分數與激勵文字
2. 將本次紀錄 push 進 `quizHistory` → `localStorage` → `syncToCloud`
3. 顯示「歷史 TOP 3」排行榜
4. 顯示「錯題檢討」清單（含「請 AI 導師講解此題」按鈕）
5. 顯示「重新測驗」與「回首頁」按鈕

## 5. AI 功能模組

### 5.1 AI 導師解析（`askAIForExplanation`）

**觸發時機：** 測驗結算頁，點擊錯題旁的「🤖 請 AI 導師講解此題」

**API 呼叫：**
```
POST https://generativelanguage.googleapis.com/v1beta/
     models/gemini-2.5-flash-lite:generateContent?key={userKey}
```

**Prompt 策略：** 以角色扮演方式請 Gemini 扮演「iPAS 考試導師」，說明為何正確答案正確、錯誤答案錯在哪裡，限制回覆 150 字以內。

### 5.2 AI 自動出題（`btnGenerateWithAI`）

**流程：**
```
使用者選擇科目 + 題數 + 情境關鍵字（選填）
        ↓
組裝嚴格格式的 Prompt（要求只輸出純 JSON 陣列，不含 markdown）
        ↓
呼叫 Gemini API（temperature: 0.7）
        ↓
清除回應中可能殘留的 ```json 標記
        ↓
JSON.parse() 解析
        ↓
進入「AI 題目審核視窗」供使用者確認
        ↓
確認後 concat 進 activeQuizDatabase → localStorage → syncToCloud
```

**金鑰儲存：** 僅存於使用者自己的 `localStorage`，不上傳至 GAS 雲端。

## 7. 列印與匯出

### 7.1 列印錯題 / 我的最愛

`printQuestions(type)` 動態組裝 `#printableArea` 的 HTML，再呼叫 `window.print()`。

CSS 中有 `@media print` 規則：
- 隱藏所有 `.app-container` 和 `.fab-container`
- 只顯示 `#printableArea`
- 每題包含題目、選項（正解標示 ✅）、解答、筆記空白區

### 7.2 匯出 JSON 備份

`handleExport(isOfficial)` 將題庫序列化為 JSON，透過動態 `<a>` 標籤下載檔案。  
檔名格式：`iPAS_Custom_DB_YYYYMMDD.json` / `iPAS_Official_{屆次}_YYYYMMDD.json`

## 9. 已知問題與注意事項

### 9.1 考試倒數日期為硬編碼

```javascript
const targetDate = new Date('2026-05-16T00:00:00').getTime();
```

考試日期已過（目前為 2026-06-10），頁面會顯示「🎉 考試已經開始囉！祝你金榜題名！」，  
**若要更新下一次考試日期，需直接修改此行。**

### 9.2 雲端同步為「取多的」策略，不支援刪除同步

若在裝置 A 清除錯題，之後在裝置 B 開啟頁面，  
`syncFromCloud()` 會把雲端還有的錯題同步回裝置 A，清除操作無法傳播到雲端。  
**若要真正清除，必須在所有裝置都執行清除，或直接清空 GAS 雲端的資料。**

### 9.3 官方題庫屆次為硬編碼

```html
<option value="114年第4次">114年第4次</option>
<option value="115年第1次">115年第1次</option>
...
```

新增考試屆次需要手動修改 HTML 中的 `<select>` 選項。

### 9.4 AI 出題依賴使用者自備金鑰

使用者必須自行至 Google AI Studio 申請 Gemini API Key，系統不提供預設金鑰。  
金鑰儲存於 localStorage，不會上傳至 GAS，安全性由使用者的裝置保護。

### 9.5 預設題庫只有 1 題

```javascript
const defaultQuizData = [
    { subject: "科目1：...", question: "...", options: [...], answer: "..." }
];
```

系統預設題庫僅包含 1 道範例題，使用者需自行匯入或使用 AI 出題功能擴充題庫。
